# Advanced Configuration

## Environment Variables

OpsOrchestra uses pydantic-settings. Configure these in your `.env` file.

| Variable | Description | Default | Required |
|----------|-------------|---------|----------|
| `ANTHROPIC_API_KEY` | API Key for Claude | - | Yes |
| `CHROMA_PERSIST_DIR` | Path to vector store | `./.chroma` | No |
| `LOG_LEVEL` | Python logging level | `INFO` | No |
| `LANGFUSE_PUBLIC_KEY` | Observability public key | - | No |
| `LANGFUSE_SECRET_KEY` | Observability secret key | - | No |

## Model Selection

Currently, the agent is optimized for **Claude 3.5 Sonnet** (`claude-3-5-sonnet-20240620`).

To change the model, modify `agent/graph.py`:

```python
# Create the LLM
llm = ChatAnthropic(
    model="claude-3-5-sonnet-20240620",
    temperature=0,
    max_tokens=4096
)
# Note: Changing to GPT-4o requires updating tool definitions.
```

## Vector Store Settings

We use ChromaDB. Performance tuning can be done in `ingestion/vectorstore.py`:

- **HNSW Index**: Adjust `ef_construction` and `M` for trade-off between recall and speed.
- **Persistence**: Data is saved to `CHROMA_PERSIST_DIR`. For production, connect to a Chroma server instance.
